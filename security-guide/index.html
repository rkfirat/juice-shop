Static Analysis with Semgrep

We use Semgrep in OSS mode for automated security scanning. It checks every commit for common vulnerabilities (OWASP Top 10), misconfigurations, and hardcoded secrets.

  • Zero configuration overhead
  • Free Open Source engine
  • Customizable YAML rules
- name: Semgrep OSS Scan
  uses: semgrep/semgrep-action@v1
  with:
    config: auto

Supply Chain Security with Syft

- name: Generate SBOM (Syft)
  uses: anchore/sbom-action@v0
  with:
    format: cyclonedx-json
    output-file: sbom.json

Syft generates a Software Bill of Materials (SBOM) for our project, providing full visibility into every library and dependency used.

  • Identify vulnerable dependencies
  • Catalog all nested modules
  • CycloneDX JSON integration

Dynamic Scanning: Nuclei & Invicti

For runtime protection, we combine Nuclei's template-based checks with Invicti's deep DAST capabilities. This catches vulnerabilities that only manifest in a running environment.

Nuclei (OSS) Invicti (Enterprise) DAST 
# Run Nuclei locally
docker run --rm projectdiscovery/nuclei \
  -u http://localhost:8080 \
  -status

Continuous Deployment to GHCR

- name: Build and push Docker image
  uses: docker/build-push-action@v5
  with:
    push: true
    tags: ghcr.io/${{ github.repository }}:latest

Our secure pipeline automatically builds and pushes the Docker image to GitHub Container Registry only after all security gates are successfully cleared.

  • Automated GHCR login
  • Immutable tagging (Commit SHA)
  • Security-gated promotion
