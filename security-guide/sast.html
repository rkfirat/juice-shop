Static Analysis (SAST)

Scanning source code for vulnerabilities using Semgrep OSS.

What is SAST?

Static Application Security Testing (SAST) involves analyzing the application's source code, byte code, or binaries for security vulnerabilities. Since it doesn't require a running system, it can be integrated early in the development cycle (Shift-Left).

Our Tool: Semgrep OSS

Semgrep is a fast, open-source static analysis engine for finding bugs and enforcing code standards. Its rules look like the code you already write.

# Example Semgrep Rule for hardcoded secrets
rules:
  - id: hardcoded-api-key
    pattern: |
      $KEY = "..."
    message: "Hardcoded API Key detected!"
    severity: ERROR
    languages: [javascript, python]

GitHub Integration

Results from our Semgrep scans are automatically uploaded to the GitHub Security Tab in SARIF format. This gives you a clear dashboard within the repo to track and resolve findings.

  • Automatic Pull Request comments.
  • Gated merges based on severity.
  • SARIF output for advanced dashboarding.
