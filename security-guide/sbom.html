Supply Chain Security (SBOM)

Tracking dependencies and ensuring transparency with Syft.

The Importance of SBOM

Modern applications are built using thousands of third-party libraries. A Software Bill of Materials (SBOM) is an inventory of every component in your software, allowing you to quickly react when a new vulnerability is discovered in one of your dependencies (e.g., Log4j).

Our Tool: Syft

Syft is a powerful tool for generating SBOMs from container images and filesystems. It supports multiple formats, including the industry-standard CycloneDX.

# Configuration in GitHub Actions
- name: Generate SBOM
  uses: anchore/sbom-action@v0
  with:
    format: cyclonedx-json
    output-file: sbom.json

CycloneDX Format

We use CycloneDX JSON format because it is machine-readable and compatible with many vulnerability scanners and compliance tools. This allows us to automate the checking of our dependencies against known CVE databases.
