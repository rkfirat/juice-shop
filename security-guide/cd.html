Continuous Deployment (CD)

Automated, secure delivery to GitHub Container Registry.

Secure Pipeline Architecture

Our CD pipeline is designed with "Security as a Gate". This means that the deployment step only triggers if all preceding security tests (SAST, SBOM, etc.) complete successfully.

GitHub Container Registry (GHCR)

We use GHCR to store our Docker images. It is fully integrated with GitHub's ecosystem, allowing for seamless authentication and permissions management using the standard GITHUB_TOKEN.

- name: Log in to GHCR
  uses: docker/login-action@v3
  with:
    registry: ghcr.io
    username: ${{ github.actor }}
    password: ${{ secrets.GITHUB_TOKEN }}

Tagging Strategy

We use a dual tagging strategy for our images:

  • Latest: Points to the most recent successful build on the main branch.
  • Commit SHA: An immutable tag that points to the exact version of the code, essential for rollbacks and traceability.
